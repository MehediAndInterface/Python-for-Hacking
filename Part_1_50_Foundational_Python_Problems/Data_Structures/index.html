The Science of Speed: Time Complexity

We measure algorithm efficiency with "Big O notation". It describes how an algorithm's runtime scales with the size of the input data. Lower is better.

Summation: O(n) - Linear Time

The `sum()` function must look at every single element once. As the list grows, the time taken grows linearly.

Length: O(1) - Constant Time

The `len()` function takes the same amount of time to execute, no matter how large the list is. This is incredibly fast.
